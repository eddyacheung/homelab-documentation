# Notes

This folder is for concept notes, study notes, and reusable learning material that does not describe the current state of the homelab.

Use this section for ideas and explanations that are useful across work, homelab projects, and future learning.

## Suggested Structure

```text
notes/
├── docker/
├── git/
├── linux/
├── networking/
├── vscode/
└── ai/
```

## Difference Between Notes and Documentation

Use `services/`, `networking/`, `docker/`, `linux/`, `windows/`, and `standards/` for documentation about the actual homelab.

Use `notes/` for concepts, explanations, and study material.

Examples:

- Why Docker bind mounts matter
- Git rebase vs merge
- VS Code workflow tips
- Linux permissions
- DNS troubleshooting patterns
- AI model notes
